# 회고 (Retrospective): 마찰을 증거로, 증거를 개선으로

> **파일명 유래**: `retrospective`(회고, 되돌아봄). 끝난 작업을 돌아보며 배운 것·반복되는 마찰을 기록해서 이 이름이다. 애자일의 스프린트 회고에서 따왔다.

> 회고는 감상문이 아니다. **반복되는 마찰을 증거로 붙잡아, 다음 개선(또는 v3)의 재료로 만드는** 장치다.
> "이번에 좀 힘들었다"로 끝나면 휘발된다. 같은 마찰이 몇 번 반복됐는지 *세어야* 신호가 된다.
> 최종 검토일: 2026-06-21.

## 왜 필요한가 (이게 텐션인 이유)

작업이 끝나면 보통 그냥 끝난다. 그러면 어디서 반복해서 막히는지가 머릿속에만 남고 휘발된다.
회고는 그 마찰을 **파일에 누적**시킨다. 같은 항목이 RETRO.md에 3번 쌓이면 ─ 그건 우연이 아니라 **구조적 문제**고, 템플릿·프로세스를 바꿔야 한다는 증거다. 막연한 기억이 아니라 셀 수 있는 증거로.

> 느슨한 회고("좋았다/아쉬웠다")는 무게만 늘리고 텐션이 없다.
> 텐션 있는 회고는 **반복을 카운트하고, 임계치를 넘으면 행동을 강제한다.**

## 언제 하나

- **마일스톤·로드맵 단계 완료 시** (권장): 큰 단위가 끝날 때.
- **주기적으로** (선택): 여러 작업이 쌓였을 때.
- 매 작업마다는 과하다 ─ 회고가 의식이 되면 형식만 남는다.

## `/retro` 절차

### 1. 이번 구간을 사실로 회고한다 (감상 아닌 사실)
- **무엇을 했나**: 완료한 작업·단계 (셀 수 있게).
- **무엇이 잘 됐나**: 반복하고 싶은 구체적 행동 (막연한 "잘됨" 금지 ─ "인터뷰에서 범위를 좁힌 게 효과적이었다"처럼).
- **무엇이 막혔나**: 어디서 멈췄나, 무엇이 어려웠나 (구체적으로).

### 2. 막힌 것을 분류한다 (반복 카운트의 핵심)
막힌 것을 RETRO.md의 **반복 마찰 표**에 기록한다. 이미 있는 항목이면 **카운트를 +1** 한다.

```markdown
## 반복 마찰 (누적 ─ 카운트가 임계치)
| 마찰 | 카운트 | 분류 | 비고 |
|------|--------|------|------|
| 검수 후 다음 결정에서 막힘 | 3 | 결정 | v2.12 결정보조로 대응 중 |
| 여러 프로젝트 오가다 맥락 잃음 | 2 | 연속성 | 로드맵 /resume으로 대응 |
```

### 3. 임계치를 넘으면 행동한다 (텐션의 핵심)
> **[MUST] 카운트 3에 도달하면 그냥 넘어가지 않는다.** 세 번 반복된 마찰은 우연이 아니라 구조적 문제다. 반드시 행동(격상)한다.

- **카운트 3 이상**: 우연이 아니다. **구조적 문제로 격상**한다 ─ 템플릿·프로세스 개선 항목으로 BACKLOG에 올리거나, 반복이 근본적이면 **v3 신호**로 표시한다. (격상 없이 그냥 카운트만 올리는 것 금지)
- **카운트 2**: 관찰 중. 한 번 더 나오면 격상.
- **카운트 1**: 일단 기록만. 일회성일 수 있다.

> 이 임계치가 회고의 텐션이다. "또 막혔네"를 그냥 넘기지 않고, **세 번째엔 반드시 행동**하게 만든다.

### 4. 다음 구간에 반영한다
- 잘 된 것 → 계속한다 (명시적으로).
- 격상된 마찰 → 개선 작업으로 만든다 (BACKLOG 또는 v3 검토).

## RETRO.md 서식

```markdown
# 회고 누적: <프로젝트명>

## <날짜> ─ <구간/마일스톤>
- 한 일: ...
- 잘 된 것 (반복할 것): ...
- 막힌 것: ...

## 반복 마찰 (누적)
| 마찰 | 카운트 | 분류 | 비고 |
|------|--------|------|------|
| ... | N | ... | ... |

## 격상된 항목 (카운트 3+ → 행동)
- <마찰>: → BACKLOG 또는 v3 신호. 대응: ...
```

저장: `docs/plans/RETRO.md` (없으면 생성, 있으면 날짜 섹션 추가 + 마찰 카운트 갱신).

## 연결

- 격상된 마찰 → `docs/plans/BACKLOG-deferred.md`(개선거리) 또는 버전 검토.
- "이 템플릿 자체의 마찰"이 반복되면 → 템플릿 개선(다음 버전)의 재료. **회고가 곧 버전 진화의 엔진이다.**
- 로드맵 단계 완료(`roadmap.md`) → 회고 제안 지점.
