# 적용 명령: agent-loop / harness-design / kit 접목

> **파일명 유래**: 이 체제(loop)를 기존 레포에 `apply`(접목)하는 `prompt`(프롬프트)라 이 이름이다. 새 프로젝트가 아닌 진행 중 프로젝트에 템플릿을 입힐 때 쓴다.

이 파일의 지시만 따라. 이 프로젝트에 행동 루프와 강제 게이트를 접목해줘.
**파일을 수정하기 전에 계획을 먼저 보여주고 내 승인을 받아.**
(import 대상 = `agent-loop.md` · 게이트 설계 참조 = `harness-design.md` · 레퍼런스 구현 = `kit/`)

**0) 읽기**
- `agent-loop.md`·`harness-design.md`·`kit/README.md`를 끝까지 읽어. (텍스트로: 스크린샷이면 정확 복사 안 됨)

**1) 자동로드 지침 파일 식별**
- 이 도구가 매 세션 자동으로 읽는 지침 파일 확인(Claude Code=루트 `CLAUDE.md` / 그 외=`AGENTS.md`·`.cursor/rules` 등).
- 없으면 만들고, 있으면 그걸 써. `docs/`에만 두지 마(사람만 본다).

**2) 레포 조사 (묻기 전에 먼저)**
코드·설정에서 직접 찾아(있는 것만):
- 테스트·린트·빌드·타입체크 명령과 "무엇을 보이면 통과인가" → `verify.sh`로 모을 수 있는지
- 파일/모듈 짝 규칙 · 배포·릴리스 절차와 확인 방법
- 이미 있는 게이트: CI 필수체크, 보호 브랜치, pre-commit, 승인 흐름, PreToolUse 훅
- 시크릿·민감정보 취급, 위험 명령(삭제·force push·배포·외부 전송)
> 조사로 알 수 있는 건 묻지 말고 채워. **진행을 막는 모호함만** 질문으로.

**3) 계획 제시 (아직 수정하지 마)**

A. **자동로드 지침 파일**
   - `agent-loop.md`는 **복사 대신 import**(예: `@agent-loop.md`). 미지원이면 핵심만 인라인. 파일은 짧게(<200줄).
   - 본문엔 매 작업 필요한 것만: 행동 MUST 요약 / **§2 검증(통과 = `./verify.sh` exit 0)** / 파일·모듈 짝 규칙 / 위험 작업 목록.
   - `harness-design.md`·`kit/`로 가는 **한 줄 포인터** + 로드 카나리 한 줄.
   - `CLAUDE.template.md` 구조를 그대로 채워 쓰는 걸 권장.

B. **강제 게이트(kit) 배선 제안**
   - `kit/verify.sh`를 2)에서 찾은 실제 검사로 채워 루트에 둘지 제안(비우면 의도적 fail = 검증 빈칸 차단).
   - `kit/hooks/guard.sh`+`settings.json`을 `.claude/`에 설치하고 deny 패턴에 이 레포 위험 명령 추가할지 제안.
   - `kit/ci.example.yml`을 이 레포 CI로 이식할지 제안.
   - 이미 있는 게이트는 **새로 만들지 말고** "있음 → 통과·존중"으로 기록. 없으면 "없음 → 규율로 강제".
   - 위 게이트 **생성·설치는 전부 내 승인 후**.

C. **harness-design.md §4 바인딩 표**: 찾은 값으로 채우고, 내가 줘야 채울 칸(배포 절차·권한 그룹·승인 경로·문서 신선도 강제)은 **빈칸 + 질문**으로.

**4) 승인 후 적용**
- 내가 OK 하면 적용. 적용 후 지침 파일 로드 확인(예: `/memory`)하고 `./verify.sh`가 실제로 도는지 확인.

**5) 보고**
- 무엇을 어디에 넣었는지 · 채운 바인딩 값 · 못 채운 칸(내가 줄 값) · 새로 만들지 않고 존중만 한 기존 게이트.
