# 자연어 의도 연결 (Intent Routing): 명령어를 몰라도 되게

> **파일명 유래**: 사용자의 `intent`(의도)를 맞는 흐름으로 `routing`(연결)한다. 명령어 이름이 아니라 평소 말로 들어온 의도를 알아채 연결해서 이 이름이다.

> 사용자는 명령어 이름으로 생각하지 않는다. **자기 상황**으로 생각한다. "버그 고치고 싶어"라고 하지 "/bugfix 써야지"라고 하지 않는다.
> 그래서 **평소 말(자연어)로 하면 AI가 맞는 흐름으로 연결**한다. 명령어를 외울 필요가 없다.
> **단, 자연어는 모호하다. 잘못 이해하고 실행하면 명령어보다 위험하다.** 그래서 "편하게 받되, 모호·위험하면 멈추고 확인"이 이 문서의 핵심이다.
> 최종 검토일: 2026-06-21.

## 1. 원칙: 자연어 우선, 명령어는 단축어일 뿐

- 슬래시 명령(`/spec` 등)은 *빠른 단축어*다. **그게 있어야 작동하는 게 아니다.**
- 사용자가 평소 한국어로 말하면, AI가 그 의도를 읽어 맞는 흐름으로 연결한다.
- 사용자에게 "그건 /bugfix로 하세요" 식으로 **명령어를 강요하지 않는다.** 그냥 그 흐름을 태우면 된다.

### 의도 → 흐름 매핑 (예시, 사용자 표현은 다양할 수 있음)
| 사용자가 이렇게 말하면 | 연결할 흐름 (명령 등가) |
|------------------------|--------------------------|
| "새로 만들고 싶어 / 시작하고 싶어" | 새 프로젝트 인터뷰 (`/init-project`) |
| "이거 만들고 싶은데 막연해" | 명세 인터뷰 (`/spec`) |
| "기능 추가하고 싶어" | 기능 작업 (`/feature`) |
| "버그 고치고 싶어 / 이게 안 돼" | 원인규명 후 수정 (`/bugfix`) |
| "다 됐는지 확인해줘 / 돌아가?" | 기술 검증 (`/verify`) |
| "이게 진짜 목표대로 됐나?" | 의미 검증 (`/validate`) |
| "막혔어 / 더 못 하겠어" | 에스컬레이션 (`/escalate`) |
| "딴 거 하다 왔어 / 어디까지 했지?" | 재개 브리핑 (`/resume`) |
| "돌아보자 / 뭐가 잘 안 됐지" | 회고 (`/retro`) |
| "이번 거 정리해서 넘겨줘" | 인수인계 (`/handoff`) |

> 이건 *예시*다. 사용자 표현은 무한히 다양하다. 표현을 기계적으로 맞추려 하지 말고, **의도를 읽어** 연결한다.

## 2. 안전장치: 잘못 이해하면? (이게 핵심)

자연어는 모호하다. AI가 잘못 알아듣고 실행하면 위험하다. 그래서 **행동하기 전에 위험도로 나눈다.** (agent-loop ②결정 원칙 위에 얹는다)

### 위험도별 행동
**낮은 위험 (되돌리기 쉬움)** → 이해한 대로 진행하되, **무엇을 했는지 보고**한다.
- 예: "이 함수 정리해줘" → 들여쓰기·변수명 정리 후 "이렇게 정리했어요" 보고.

**모호함 (여러 해석 가능)** → **추측으로 실행하지 않고 되묻는다.** (환각으로 메우지 않기)
- 예: "이거 정리해줘" → "어떤 정리를 원하세요? 1)코드 리팩토링 2)안 쓰는 파일 삭제 3)문서 정돈"
- 이게 "잘못 이해하면?"의 1차 방어선이다. 모호하면 *멈춘다.*

**높은 위험 (비가역: 삭제·배포·되돌리기·외부전송)** → **반드시 "이해한 것 되말하고 확인" 후 실행.**
- 예: "그거 지워" → "이렇게 이해했어요: [파일 X 3개 삭제]. 되돌릴 수 없어요. 진행할까요?" → 승인 전 실행 금지.
- agent-loop ②의 "비가역은 계획→확인→실행"과 동일. 자연어일수록 이 확인이 더 중요하다.

### 가장 강한 안전장치: 이해한 것을 되말하기 (Echo-back)
실행 전에 **AI가 이해한 바를 사람 말로 되비춘다.** 그래야 잘못 이해했어도 *행동 전에* 사용자가 잡아낸다.
- 예: "어제 하던 거 이어서" → "어제 PLAN-A2(로그인) 만들다 멈추셨고, 다음은 비번 검증부터예요. 이어서 할까요?"
- 잘못된 *실행*이 아니라 잘못된 *이해*에서 멈추게 한다. 비용이 가장 싼 지점에서 교정.

## 3. 능동 안내: 묻기 전에 길을 깔아준다

사용자가 "그거 뭐였지"에 빠지기 전에, AI가 작업 흐름의 분기점에서 **다음 할 일을 사람 말로 먼저 제안**한다. (명령어 이름이 아니라 행동으로)
- 작업을 막 끝냈으면 → "이제 검증할 차례예요. 확인해드릴까요?"
- 검증 통과했으면 → "통과했어요. 다음은 배포할지 / 다음 기능으로 갈지 결정할 차례예요." (decision-support)
- 다른 일 하다 돌아온 낌새면 → "이어서 하시려면 지금까지 한 걸 정리해드릴게요."
- 단, 과하지 않게. 사용자가 "그만 제안해"라고 하면 멈춘다.

## 4. "뭐 할 수 있어?"라고 물으면

명령어 목록을 나열하지 않는다(사용자는 그 뜻을 모른다). **상황으로 답한다:**
```
지금 무엇을 하고 싶으세요? 평소 말로 하시면 돼요:
 · 새로 만들기 → "~ 만들고 싶어"
 · 만든 거 확인 → "확인해줘"
 · 안 되는 거 고치기 → "이게 안 돼"
 · 막혔을 때 → "막혔어"
 · 다른 일 하다 왔으면 → "어디까지 했지?"
```
명령어를 외우게 하지 말고, **"이럴 땐 이렇게 말하면 된다"**를 알려준다.
