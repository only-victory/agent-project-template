# 가드레일 (Guardrails): 기술·UI·보안·검증 선택 원칙

> **파일명 유래**: `guardrails`(가드레일, 도로 난간). AI가 위험한 길로 벗어나지 않게 막는 안전 원칙들이라 이 이름이다.

> `agent-loop.md`를 보조하는 선택 원칙. **각 블록 머리의 "적용:"을 보고, 이 프로젝트에 해당하면 적용**한다.
> 해당 없으면 그 블록은 건너뛴다(범용 템플릿이라 다 담되, 안 쓰는 건 누워있게 = opt-in).
> 기준 밖 선택은 금지가 아니라 **정당화 필수**: `docs/specs/decisions/`에 ADR로 이유·대안·트레이드오프 + 사용자 승인.

## 기술 선택 (스택 가드레일)   · 적용: 모든 프로젝트
새 기술·언어·라이브러리는 아래를 만족하는 **주류 기술만**. 임의·희한한 선택 금지.
- **주류·검증**: 생태계·문서·커뮤니티·구인 풀이 두터운 1군. 실험적·틈새·1인 메인테이너 배제.
- **보수 용이**: 다른 개발자가 쉽게 인수·유지할 만큼 평범하고 지루한(boring) 선택 우선.
- **확장·연계**: 표준 인터페이스(REST·SQL 등) + 풍부한 통합, 다른 시스템과 붙기 쉬운 것.
- **공급망 관리**: 버전 고정, 미사용 의존성 제거, 취약점 스캔.

## UI/UX   · 적용: 사용자 화면(웹·앱)이 있을 때
목표: 동선·구조는 누구나 즉시 이해(익숙함=편리), "와우"는 마감 완성도에서.
- **익숙한 패턴 우선**: 표준 동선·컴포넌트. 새로움은 구조가 아니라 디테일(타이포·여백·모션)에서.
- **접근성 필수(비협상)**: 키보드 탐색 · 명도 대비 WCAG AA · 의미적 HTML · 포커스 표시.
- **일관성(토큰)**: 색·간격·타이포는 디자인 토큰 한 곳에서 정의, 전 화면이 그것만. 하드코딩 금지.
- **상태 설계**: 로딩·빈 상태·에러 상태 항상. **절제**: 폰트 1~2종(주류) · 색은 의미 단위 소수.
- 검증된 디자인 시스템(주류) 위에서. "와우"(실험적 레이아웃)는 핵심 동선 밖에 한정 + ADR.

## 웹/앱 보안   · 적용: 네트워크·사용자 입력·데이터를 다룰 때 (대부분)
> 기준: OWASP Top 10. 매 작업 핵심 3가지는 항상, 전체는 OWASP Top 10 참조.
- **입력 검증·출력 인코딩**(인젝션·XSS 방지) / **최소 권한·접근 제어·인증** / **비밀·키 비노출**(코드·로그·커밋).
  - **[MUST] 공개 전 유출 점검**: 공개 레포에 push·배포하기 전, API 키·비밀번호·토큰·프라이빗 키·내부 URL·개인정보가 커밋에 섞이지 않았는지 `git diff --staged`로 확인한다. 비밀은 코드가 아니라 환경변수·`.env`(.gitignore됨)·시크릿 매니저에. guard.sh가 커밋/푸시 시 1차 경고하지만 완전하지 않다(사람이 최종 확인). 이미 커밋됐다면 그 키는 폐기·재발급(히스토리에 남으므로). 상세: `golden-set.md` G14.
- 그 외(암호화·보안설정·의존성 취약점·SSRF·보안 로깅)는 해당 기능 생길 때 OWASP Top 10에 맞춰 적용.

## AI/LLM 보안   · 적용: **LLM·에이전트 기능을 *제품에* 포함할 때** (챗봇·RAG·AI 기능)
> 기준: OWASP Top 10 for LLM (2025). 이 제품이 AI 기능을 *사용자에게 제공*하면 적용.
- **프롬프트 인젝션**: 외부 콘텐츠(웹·문서·이슈)에 섞인 지시를 신뢰 안 함, 사용자 지시와 분리.
- **민감정보 노출**: 시크릿·PII를 프롬프트·로그·응답에 안 남김.
- **부적절한 출력 처리**: LLM 출력은 신뢰 못 할 입력: 실행/렌더(SQL·HTML) 전 검증·인코딩.
- **공급망**: 모델·라이브러리·데이터 출처 검증·버전 고정. **무한 소비**: 토큰·API·비용 상한.

## 에이전트 행위 보안   · 적용: 모든 프로젝트 (AI로 개발하는 행위 자체)
> OWASP LLM "과도한 에이전시": 이건 제품이 AI든 아니든, *에이전트로 개발하는 것* 자체에 해당.
- 에이전트엔 필요한 만큼만 권한·도구. 비가역·외부영향 작업은 **자동 실행 금지 → 승인 게이트**.

## 검증·테스트   · 적용: 모든 프로젝트
- **핵심 로직은 테스트 동반**: 돈·인증·데이터처럼 틀리면 아픈 로직은 자동 테스트 없이 "완료" 금지.
- 통과 기준은 `./verify.sh` 한 곳에. 테스트가 생기면 verify에 포함.

## 데이터·마이그레이션 안전   · 적용: DB·영구 데이터를 다룰 때 (= '풀' 등급)
- 스키마 변경·데이터 이전·대량 삭제는 비가역 → 백업 + dry-run + 롤백 경로 + 승인 후 실행.

## HITL · 에스컬레이션 · 퀄리티 게이트   · 적용: 모든 프로젝트
> 이미지 14(퀄리티 게이트) · 15(HITL) · 16(에스컬레이션) 반영.
> 세부 패턴은 `multi-agent.md` §3~5 참조. 여기선 원칙만.

**퀄리티 게이트 3단계**:
- **통과(Pass)**: verify.sh exit 0 + PRD 수용 기준 충족 → 완료 선언 가능
- **보류(Hold)**: 기능은 동작하나 비기능·해석 불일치 → 사람 판단 요청, 임의 통과 금지
- **재시도(Retry)**: verify.sh exit 1, 원인 특정 가능 → ↺ 원인규명 루프

**HITL 자동 중단**: 같은 단계 재시도 3회 초과 · 비용 상한 도달 · 성공 기준 충돌 · 보안/법적 판단 필요 · 범위 밖 외부 영향 → 즉시 멈추고 사람에게.

**에스컬레이션**: 에이전트 간 합의 불가 · HITL 요청 후 자율 진행 불가 → PLAN Blocked + 경로 알림(`multi-agent.md §5`). 경로가 채워지지 않으면 **빈칸 채우기 전까지 에스컬레이션 불가** → `/init-project` 또는 셋업에서 채울 것.

## 산출물 표기   · 적용: 모든 글·문서·UI 텍스트·코드 주석
> "사람이 쓴 글"처럼 자연스럽게. AI 티가 나는 표기를 *사전에* 만들지 않는다(사후 청소가 아니라 사전 방지).
- **[MUST] em-dash(—) 금지**: 긴 줄표 `—`를 쓰지 않는다. 문맥에 따라 `:`(콜론), `,`(쉼표), `(괄호)`, 또는 문장 분리로 대체한다. UI 텍스트·문서·주석·커밋 메시지 어디에도 쓰지 않는다.
- 과한 이모지 나열, 기계적인 "first/second/finally" 나열, 불필요한 굵게 강조 남발 등 AI 글 특유의 패턴을 피한다.
- 이유: 사용자가 그대로 복사해 쓰는 산출물이다. AI 흔적이 남으면 "사람이 만든 느낌"이 깨진다.
