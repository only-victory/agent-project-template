# unattended-run: 무인 실행 (밤샘·자리 비움) 가이드

> **파일명 유래**: unattended(지켜보지 않는) run(실행). 자야 하는데 작업은 돌리고 싶을 때의 절차.
> **왜 필요한가**: 무인 루프의 3대 부채 ─ ①검증 부채(증거 없이 "완료" 주장) ②이해 부채(내가 안 쓴 코드가 이해 속도보다 빨리 쌓임) ③인지적 저항(결과를 판단 없이 수용). 무인 실행은 이 셋을 가속한다. 금지가 답이 아니라, **부채를 아침에 갚는 절차**가 답이다.

## 1. 시작 전 (자기 전 5분)

- [ ] **[MUST] 범위가 좁고 명확한 PLAN**: "막연하면 실행 금지"가 무인일 땐 더 강하다. 밤새 헤맬 여지를 남기지 마라.
- [ ] **[MUST] 안전한 작업만**: 리팩터링·테스트 작성·문서화·격리된 기능 구현처럼 **되돌리기 쉬운 것만**. 배포·DB 마이그레이션·삭제·결제·외부 전송(비가역)은 무인 금지 ─ 낮에 사람과.
- [ ] **[MUST] `--dangerously-skip-permissions` 금지**: 승인(ask)은 열어도 되지만, 이 플래그는 guard.sh·deny(차단층)까지 꺼버린다. 차단층은 당신이 자는 동안에도 깨어 있어야 한다.
- [ ] **[SHOULD] 자주 커밋 지시**: "작업 단위마다 커밋해라"를 PLAN에 명시. 아침에 되돌릴 지점이 생긴다.
- [ ] (선택) stop-verify-gate 훅 켜기: 검증 없이 마무리하는 것을 기계가 막는다. 단 무한 재시도로 밤을 새울 수 있으니 익숙해진 뒤에.

## 2. 아침 리커버리 (3단, 순서대로)

1. **검증**: `sh verify.sh` + 골든셋. 밤새 돌았다 ≠ 검증됐다.
2. **백지 검증**: "verifier로 어젯밤 작업을 PLAN 대비 검증해줘" ─ 밤샘 세션의 자기 채점을 믿지 않는다.
3. **[MUST] 설명 받기 (teach-back, 이해 부채 상환)**: "어젯밤 만든 것의 구조와 핵심 결정을 나에게 설명해줘. 파일 지도부터." 듣고 이해 안 되는 부분은 **넘어가지 말고** 다시 묻거나 단순화를 요청한다.
   > 원칙: **설명 못 듣겠는 코드는 아직 내 것이 아니다.** 이해 없이 쌓인 코드는 다음 결정을 남(AI)에게 넘기게 만든다(인지적 저항). 이해가 안 되면 리팩터링·주석 추가를 요청하고, 그래도 안 되면 그 부분은 되돌린다.

## 3. 틀어졌을 때

- 커밋 단위로 되돌린다(자주 커밋 지시가 여기서 값을 한다). 밤새 결과를 "아까워서" 끌어안지 마라 ─ 이해 못 한 채 안는 코드가 제일 비싸다.
