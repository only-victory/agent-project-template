# 하네스 설계 (Harness / Enforcement Design)

> **파일명 유래**: `harness`(하네스, 안전벨트·마구). AI를 틀 안에 묶어 통제하는 강제 장치(게이트)의 설계라 이 이름이다. 엔지니어링 4단계 중 '하네스 엔지니어링'에서 따왔다.

> **독자: 시스템·조직을 설계하는 사람.** "에이전트(또는 사람)가 따라야 할 규칙을 *무엇으로 강제할까*".
> 작업 중 에이전트의 행동 지침은 별도 문서 → **`agent-loop.md`**.
> 핵심 구분: `agent-loop.md`는 게이트를 **통과·존중**하는 쪽, 이 문서는 게이트를 **만드는** 쪽이다.
> 📦 **레퍼런스 구현: `kit/`**: verify.sh(단일 검증 진입점)·PreToolUse 가드 훅·CI 어댑터. 정책이 아니라 *실행되는 게이트*의 복붙 가능한 출발점.
>
> 최종 검토일: 2026-06-17.

---

## 0. 명제: 정책 × 강제 = 하네스

- `agent-loop.md`(정책)는 "지켜지길 바라는 것"이다. **자동으로 강제하는 메커니즘**(테스트·린트·CI·권한 게이트·실행 훅·dry-run·관측)과 짝일 때만 실제로 지켜진다.
- 강제 없이 문서만 두면 **"가짜 안전감"**이 생긴다. 핵심 항목은 *문서 권고*가 아니라 *실행되는 게이트*로 내려야 한다.
- **특히 주의**: `agent-loop.md`를 에이전트 지침 파일(CLAUDE.md·AGENTS.md 등)에 넣는 건 *강제가 아니다*. 도구는 그 파일을 **컨텍스트**로 읽을 뿐이다. → §2.5.

---

## 1. 강제 강도 = f(가역성 · blast radius)

강제의 세기는 "루프 단계"가 아니라 **되돌릴 수 있는가 / 틀리면 얼마나 나쁜가**로 정한다.

| 강도 | 적용 대상 (가역성·영향) | 성격 |
|---|---|---|
| **권고(Advisory)** | 읽기 전용 / 로컬·즉시 되돌릴 수 있는 변경 | 가이드만. 어겨도 막지 않음 |
| **자동검사(Check)** | 자동으로 돌되 통과를 강제하진 않음(예: 로컬 린트·테스트) | 실패 시 경고(진행 가능) |
| **차단(Block)** | **공유 코드 머지 · 비가역 · 프로덕션 · 실데이터 · 외부 전송** | 통과 못 하면 진행 불가 + 비가역은 **수동 승인** 추가 |

> 원칙: 항목마다 "권고/검사/차단"을 정하고 **가능한 한 차단 쪽으로 올린다.**
> **공유 코드 머지·비가역 작업은 '차단(자동)'이 기본.** '검사(경고)'·'권고'에 두면 설계 결함.

---

## 2. 단계별 강제 장치 룩업 (무엇을 자동화할지)

| 지점 | 자동검사(Check) 후보 | 차단(Block) 후보 |
|---|---|---|
| 변경 통제 | PR·코드리뷰 권장 | 보호 브랜치(required review), 규칙의 코드화(OPA / pre-commit·CI 훅) |
| 실행(사전) | 린터·포매터·타입체크·빌드, 시크릿/취약점 스캔 | CI에서 위 실패 시 머지 차단, 최소권한·토큰 스코프·샌드박스 |
| 검증 | 단위/통합/회귀 테스트, 증거 출력 | **CI 필수 체크**(= `verify.sh`를 CI에서 실행, 미통과 시 배포 차단), 증거 없으면 '완료' 불가 |
| 가역성·배포 | dry-run/--check, 스테이징 검증 | 프로덕션 분리, 카나리/점진 배포, 배포 후 헬스체크→자동 롤백, **비가역 작업 수동 승인** |
| 멱등성·동시성 | 재실행 안전 점검 | idempotency key, 락/트랜잭션, unique 제약 |
| 관측·감사 | 로그·메트릭 확인 | 감사 추적 필수 기록, 알림 임계치 |
| 중단·한계(에이전트) | 셀프체크·검증 서브에이전트 | 최대 재시도/타임아웃/비용 상한 도달 시 자동 중단·에스컬레이션 |

## 2.5 에이전트 도구에서의 강제: 지침 파일 ≠ 게이트

에이전트 지침 파일(`CLAUDE.md`·`AGENTS.md`·`.cursor/rules` 등)은 매 세션 *읽는* **컨텍스트**이지 강제 설정이 아니다. 모델 판단과 무관하게 **행동 자체를 차단**하려면 도구의 실행 훅을 쓴다.

| 강도 | 에이전트 도구에서의 수단 |
|---|---|
| **권고(Advisory)** | 지침 파일 본문: 읽지만 강제 안 됨 |
| **차단(Block)** | **PreToolUse 훅**(`exit 2`만 차단 유효: `exit 1`은 경고일 뿐 통과됨), CI 필수 체크. 레퍼런스: `kit/hooks/guard.sh` |
| **감사(Audit)** | 지침 로드 로깅 훅, 실행 로그 |

> `agent-loop.md`의 **자동강제 가능한 [MUST]**(비가역 확인·시크릿 비노출·위험 명령 가드)는 *문장*이 아니라 **PreToolUse 훅**으로 내려야 실제 게이트가 된다.
> 단, 판단·정직성처럼 훅으로 못 막는 건 권고로 남기고 보고·리뷰로 감시한다.

---

## 3. 가장 강한 골격 (최소 4종)

1. **CI 필수 체크**: `verify.sh`를 CI에서 실행, 미통과 시 머지/배포 차단. (`kit/ci.example.yml`)
2. **보호 브랜치 / 변경 통제**: 직접 push 금지, 리뷰 필수.
3. **비가역 작업의 수동 승인**: 삭제·프로덕션·실데이터·외부 전송.
4. **(에이전트 사용 시) PreToolUse 가드**: 위험·비가역 명령을 모델 판단과 무관하게 차단. (`kit/hooks/guard.sh`)

## 4. 프로젝트 바인딩: 보편 개념 → 이 레포의 구체값

두 문서·kit은 그대로 두고, 프로젝트별로 아래만 채워 **실제 강제 장치에 연결**한다.

| 보편 개념 | 바인딩할 구체값 (채워야 작동) |
|---|---|
| 검증(증거 기준) | **`verify.sh`에 이 레포 검사 채우기**(테스트·린트·타입·빌드 + "무엇을 보이면 통과인가"). 비우면 의도적으로 fail → `agent-loop` ④ MUST가 즉흥 검증으로 변질되는 것 차단. CLAUDE.md §2도 동일하게. |
| 관련 요소 함께 보기 | 이 레포의 파일/모듈 짝 규칙 |
| 부수효과 확인 | 배포·릴리스 절차, 확인 방법 |
| 권한·감사 | 권한 그룹/가드, 감사 로그 위치 |
| 가역성 게이팅 | 위험 작업 목록 + dry-run/롤백 수단 |
| **에이전트 MUST의 강제(자동화 가능분)** | `kit/hooks/guard.sh`의 deny 패턴에 이 레포 위험 명령 추가 + `.claude/settings.json` 등록. 훅 미사용이면 "없음 → 규율로 강제" 명시 |
| 중단·에스컬레이션 | 조직의 승인·에스컬레이션 경로 |
| **문서 신선도(강제)** | 갱신을 릴리스/CHANGELOG/주기 리뷰에 묶기 + `agent-loop` ⑤ 자가치유 MUST(불일치 발견 시 diff 제안)로 상시 보강 |

> 요약: **골격(문서)은 보편, 알맹이(바인딩)는 프로젝트별.** 바인딩이 *실행되는 게이트*(CI·보호 브랜치·승인·PreToolUse 훅)에 연결될 때 비로소 "하네스"가 된다. 표가 비면 그냥 권고 모음이다.

> **권한(allow/ask/deny) = 1차 게이트, guard.sh(PreToolUse) = 2차망.** 분류 기준·조절법은 `.claude/PERMISSIONS.md`가 본체(중복 설명 안 함).
> **업계 어휘와의 연결 (네 층 어휘)**: 이 템플릿은 네 층을 모두 구현한다. ① 프롬프트 엔지니어링(지시 한 번을 잘 쓰는 법) = 슬래시 명령·인터뷰(막연한 말→정밀 명세). ② 컨텍스트 엔지니어링(AI가 무엇을 보게/안 보게 하나) = CLAUDE.md 자동 로드·기억=파일·session-start 주입·/compact, 그리고 verifier의 격리(안 보게 하는 것도 설계). ③ 하네스 엔지니어링(모델 밖의 기계 장치) = 권한·훅·게이트·verify 스크립트(AI가 잊어도 결정적으로 작동). ④ 에이전틱 엔지니어링(자율 루프로 일하는 방식 전체) = 루프 5단계·서브에이전트 오케스트레이션·HITL·완료 계약. 넓게는 ②~④ 전체를 하네스라 부르기도 한다. 이 템플릿의 입장: 신뢰성의 원천은 ①이 아니라 ②~④다.
> **훅 라이프사이클 = 지침의 기계 승격**: 문서 [MUST]는 어겨질 수 있지만 훅은 결정적이다. 이 템플릿의 승격 현황: 위험명령·비밀유출(PreToolUse, 켜짐) · 세션 위치 복원(SessionStart, 켜짐) · PLAN 날짜 갱신(PostToolUse, 켜짐) · 검증 없이 마무리 차단(Stop, 꺼짐) · 토큰 실측 기록(SessionEnd, 꺼짐) · 인터뷰 중 코드 수정(명령 allowed-tools 잠금). 안내: `.claude/hooks/HOOKS.md`.
> **검증 독립성도 강제 장치다**: 구현자와 채점자를 분리(`verifier` 서브에이전트, 읽기 전용)하면 "자기 채점"의 구조적 맹점을 메커니즘으로 막는다.
