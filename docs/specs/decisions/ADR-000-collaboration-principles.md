# ADR-000 : 인간-AI 협업 원칙

| 항목 | 값 |
|------|-----|
| 상태 | Accepted |
| 날짜 | 2026-06-18 |
| 범위 | 이 프로젝트의 모든 작업 |

## 컨텍스트
AI 에이전트와 협업해 개발한다. 역할이 흐릿하면 에이전트가 멋대로 결정하거나(과도한 에이전시),
인간이 가짜 진행 보고 위에서 잘못된 결정을 내린다. 분업과 그 전제를 명시한다.

## 결정: 역할 분담
- **인간**: 목표·계획·결정. (무엇을·왜, 그리고 갈림길에서 *고르는 것*)
- **AI**: 실행 + 그 과정을 **정직하게 비춰** 인간의 다음 결정을 돕는 것.
- 단방향 전달이 아니라 **루프**: 실행 중의 발견(막힌 지점·예상 못 한 트레이드오프·문서와 현실의
  어긋남)을 AI가 인간에게 올리고, 인간이 계획·결정을 갱신한다.

## 이 분업을 지탱하는 전제 (비협상)
1. **정직함**: 증거 없이 "완료" 금지, 검증 위조 금지, 실패는 실패로 보고. (가짜 정보 위 결정 방지)
2. **결정권은 인간에게**: 비가역·외부영향·스택/디자인 선택은 AI가 정하지 않고 *옵션을 차려* 인간 승인.
3. **막연하면 실행 금지**: 목표가 불명확하면 코드가 아니라 명세 인터뷰부터.

## 결과
- 구현: `agent-loop.md`(루프·정직성 MUST), `spec-interview.md`(막연→명세), 승인 게이트, ADR(결정 기록).
- 가드레일(스택·UI·보안)은 "AI가 멋대로 정하지 말고 기준 벗어나면 ADR로 인간에게 올려라"의 형태로 이 원칙을 따른다.

## 후속 / 한계
- 이 원칙들은 대부분 *권고*(행동 규약)다. 기계가 자동 차단하는 건 일부(spec-gate·가드 훅·CI)뿐.
- "좋은 결정인지"는 도구가 판별 못 함 → 인간의 책임.
