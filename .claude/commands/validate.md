지금 완료한 작업이 진짜 목표를 달성했는지 골든셋으로 확인해.
docs/ops/golden-set.md 절차대로:

1. **먼저 작업 유형을 판정해** (검증 라우팅 1단계): 자동화/UI/다역할/API/멀티에이전트/버그 중 어디에 해당하나? 여러 개면 다 적용. 판정 결과를 결과 표 맨 위에 "판정 유형: ___"으로 명시.
2. G1~G3 필수 3문을 순서대로 검사.
3. **판정한 유형의 의무 문항을 하나도 건너뛰지 말고** 검사 (예: UI면 G6-G7, API면 G8-G9). 해당 없는 문항은 "해당 없음: 이유"로 명시 (침묵 스킵 금지).
4. 결과를 golden-set.md 서식대로 표로 정리.
5. 최종 판정(통과/보류/재시도) + 근거 명시.

판정이 보류 또는 재시도면 → 사람 확인 요청하고 대기.
통과면 → verify.sh 출력과 함께 ⑤ 보고로 이어간다. 그리고 "다음 결정은 이것"까지 연결(decision-support).
