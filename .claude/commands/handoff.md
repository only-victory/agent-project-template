지금까지 작업한 내용을 다음 세션 또는 다른 에이전트가 이어받을 수 있도록 인수인계 문서를 만들어.

docs/ops/memory-context.md의 HANDOFF.md 서식대로:
- 현재 상태 (완료/미완료/막힌 지점)
- 다음 세션 진입점 (어디서 시작할지)
- 열린 질문 (사람 결정 필요한 것)

저장 위치: docs/plans/HANDOFF.md (없으면 생성, 있으면 날짜 섹션 추가)
저장 후 "인수인계 완료" + 파일 경로 보고.

다른 PC로 이어갈 거면(멀티 디바이스, memory-context §2-1):
- roadmap "현재 위치"·PLAN 상태도 갱신됐는지 확인.
- 이 상태 파일들은 코드와 함께 commit+push 돼야 다음 PC가 받는다고 사용자에게 안내.
- 단, 커밋·푸시는 사용자가 직접 한다(권한 ask). "커밋하고 푸시할까요?"로 확인받아 진행하거나, 사용자가 직접 하도록 남겨둔다.
