$ARGUMENTS: 빈/새 프로젝트를 시작하려는데 막연할 수 있어. docs/ops/spec-interview.md 절차대로 진행해.

1. 인터뷰 깊이를 물어봐. 제품 시작이므로 기본 [깊게](11축) 권장.
2. **고른 깊이의 모든 축을 끝까지** 소크라테스식으로 한 번에 하나씩 캐물어:
   - 매 질문 앞에 [축 N/총축수] 진행 표시를 붙여 (예: "★1축 진짜 목표 (1/11)").
   - **모든 질문을 선택형으로** 제시해 (AskUserQuestion 도구 사용): 가능한 답 2~4개 보기 + 항상 마지막에 "직접 입력" 옵션.
     사용자가 화살표로 고르고 Enter로 답하게 해. 무조건 자유서술로만 묻지 마.
     (진짜 목표·가치 축도 보기를 주되, "직접 입력"을 눈에 띄게 둬서 자기 말로 쓰게 유도해.)
   - 각 답변 후 "지금까지 정리: ..."를 한 줄로 갱신해 보여줘.
   - **역할이 2개 이상이면**(예: 이용자+관리자, 관리자+매니저 등) 각 역할마다 여정(Step)·화면을 따로 캐물어 PRD §2-1에 역할별로 정리해.
   - 사용자가 "앞으로 돌아가자"고 하면 그 축으로 돌아가 고쳐.
   - **입문자 배려**: 전문 용어엔 괄호 풀이를 달고, 어려운 질문엔 "모르면 '모름'이라 답해도 됩니다(가정으로 채움)"를 덧붙여. "처음이에요/잘 몰라요" 신호엔 깊이를 낮추고 더 쉽게 설명해.
3. **중요: 모든 축을 다 묻기 전에는 절대 코드·파일 생성을 시작하지 마.**
   "충분한 것 같다"는 판단으로 중간에 개발로 넘어가는 것 금지.
   한 축이라도 건너뛰려면 "이 축 건너뛸까요?"라고 먼저 물어.
   (사용자가 "그만/네가 정해"라고 명시적으로 중단할 때만 예외.)
4. 다 모이면 docs/specs/PRD.md를 채우고, 인터뷰 완료 리포트(깊이·체크리스트·완성도 점수)도 적어.
5. "이대로 맞나" 확인받은 뒤 멈춰. 확인 전에는 코드 짓지 마.
